22  /***
23   * This class implements rotations in a three-dimensional space.
24  
25   * <p>Rotations can be represented by several different mathematical
26   * entities (matrices, axe and angle, Cardan or Euler angles,
27   * quaternions). This class presents an higher level abstraction, more
28   * user-oriented and hiding this implementation details. Well, for the
29   * curious, we use quaternions for the internal representation. The
30   * user can build a rotation from any of these representations, and
31   * any of these representations can be retrieved from a
32   * <code>Rotation</code> instance (see the various constructors and
33   * getters). In addition, a rotation can also be built implicitely
34   * from a set of vectors and their image.</p>
35   * <p>This implies that this class can be used to convert from one
36   * representation to another one. For example, converting a rotation
37   * matrix into a set of Cardan angles from can be done using the
38   * followong single line of code:</p>
39   * <pre>
40   * double[] angles = new Rotation(matrix, 1.0e-10).getAngles(RotationOrder.XYZ);
41   * </pre>
42   * <p>Focus is oriented on what a rotation <em>do</em> rather than on its
43   * underlying representation. Once it has been built, and regardless of its
44   * internal representation, a rotation is an <em>operator</em> which basically
45   * transforms three dimensional {@link Vector3D vectors} into other three
46   * dimensional {@link Vector3D vectors}. Depending on the application, the
47   * meaning of these vectors may vary and the semantics of the rotation also.</p>
48   * <p>For example in an spacecraft attitude simulation tool, users will often
49   * consider the vectors are fixed (say the Earth direction for example) and the
50   * rotation transforms the coordinates coordinates of this vector in inertial
51   * frame into the coordinates of the same vector in satellite frame. In this
52   * case, the rotation implicitely defines the relation between the two frames.
53   * Another example could be a telescope control application, where the rotation
54   * would transform the sighting direction at rest into the desired observing
55   * direction when the telescope is pointed towards an object of interest. In this
56   * case the rotation transforms the directionf at rest in a topocentric frame
57   * into the sighting direction in the same topocentric frame. In many case, both
58   * approaches will be combined, in our telescope example, we will probably also
59   * need to transform the observing direction in the topocentric frame into the
60   * observing direction in inertial frame taking into account the observatory
61   * location and the Earth rotation.</p>
62  
63   * <p>These examples show that a rotation is what the user wants it to be, so this
64   * class does not push the user towards one specific definition and hence does not
65   * provide methods like <code>projectVectorIntoDestinationFrame</code> or
66   * <code>computeTransformedDirection</code>. It provides simpler and more generic
67   * methods: {@link #applyTo(Vector3D) applyTo(Vector3D)} and {@link
68   * #applyInverseTo(Vector3D) applyInverseTo(Vector3D)}.</p>
69  
70   * <p>Since a rotation is basically a vectorial operator, several rotations can be
71   * composed together and the composite operation <code>r = r<sub>1</sub> o
72   * r<sub>2</sub></code> (which means that for each vector <code>u</code>,
73   * <code>r(u) = r<sub>1</sub>(r<sub>2</sub>(u))</code>) is also a rotation. Hence
74   * we can consider that in addition to vectors, a rotation can be applied to other
75   * rotations as well (or to itself). With our previous notations, we would say we
76   * can apply <code>r<sub>1</sub></code> to <code>r<sub>2</sub></code> and the result
77   * we get is <code>r = r<sub>1</sub> o r<sub>2</sub></code>. For this purpose, the
78   * class provides the methods: {@link #applyTo(Rotation) applyTo(Rotation)} and
79   * {@link #applyInverseTo(Rotation) applyInverseTo(Rotation)}.</p>
80  
81   * <p>Rotations are guaranteed to be immutable objects.</p>
82  
83   * @version $Id: Rotation.java 1705 2006-09-17 19:57:39Z luc $
84   * @see Vector3D
85   * @see RotationOrder
86  
87   */
88

100   /*** Build a rotation from the quaternion coordinates.
101    * <p>A rotation can be built from a <em>normalized</em> quaternion,
102    * i.e. a quaternion for which q<sub>0</sub><sup>2</sup> +
103    * q<sub>1</sub><sup>2</sup> + q<sub>2</sub><sup>2</sup> +
104    * q<sub>3</sub><sup>2</sup> = 1. If the quaternion is not normalized,
105    * the constructor can normalize it in a preprocessing step.</p>
106    * @param q0 scalar part of the quaternion
107    * @param q1 first coordinate of the vectorial part of the quaternion
108    * @param q2 second coordinate of the vectorial part of the quaternion
109    * @param q3 third coordinate of the vectorial part of the quaternion
110    * @param needsNormalization if true, the coordinates are considered
111    * not to be normalized, a normalization preprocessing step is performed
112    * before using them
113    */
114   public Rotation(double q0, double q1, double q2, double q3,
115                   boolean needsNormalization) {
116 
117     if (needsNormalization) {
118       // normalization preprocessing
119       double inv = 1.0 / Math.sqrt(q0 * q0 + q1 * q1 + q2 * q2 + q3 * q3);
120       q0 *= inv;
121       q1 *= inv;
122       q2 *= inv;
123       q3 *= inv;
124     }
125 
126     this.q0 = q0;
127     this.q1 = q1;
128     this.q2 = q2;
129     this.q3 = q3;
130 
131   }

133   /*** Build a rotation from an axis and an angle.
134    * <p>We use the convention that angles are oriented according to
135    * the effect of the rotation on vectors around the axis. That means
136    * that if (i, j, k) is a direct frame and if we first provide +k as
137    * the axis and PI/2 as the angle to this constructor, and then
138    * {@link #applyTo(Vector3D) apply} the instance to +i, we will get
139    * +j.</p>
140    * @param axis axis around which to rotate
141    * @param angle rotation angle.
142    * @exception ArithmeticException if the axis norm is null
143    */
144   public Rotation(Vector3D axis, double angle) {
145 
146     double norm = axis.getNorm();
147     if (norm == 0) {
148       throw new ArithmeticException("null norm");
149     }
150 
151     double halfAngle = -0.5 * angle;
152     double coeff = Math.sin(halfAngle) / norm;
153 
154     q0 = Math.cos (halfAngle);
155     q1 = coeff * axis.getX();
156     q2 = coeff * axis.getY();
157     q3 = coeff * axis.getZ();
158 
159   }

161   /*** Build a rotation from a 3X3 matrix.
162 
163    * <p>Rotation matrices are orthogonal matrices, i.e. unit matrices
164    * (which are matrices for which m.m<sup>T</sup> = I) with real
165    * coefficients. The module of the determinant of unit matrices is
166    * 1, among the orthogonal 3X3 matrices, only the ones having a
167    * positive determinant (+1) are rotation matrices.</p>
168 
169    * <p>When a rotation is defined by a matrix with truncated values
170    * (typically when it is extracted from a technical sheet where only
171    * four to five significant digits are available), the matrix is not
172    * orthogonal anymore. This constructor handles this case
173    * transparently by using a copy of the given matrix and applying a
174    * correction to the copy in order to perfect its orthogonality. If
175    * the Frobenius norm of the correction needed is above the given
176    * threshold, then the matrix is considered to be too far from a
177    * true rotation matrix and an exception is thrown.<p>
178 
179    * @param m rotation matrix
180    * @param threshold convergence threshold for the iterative
181    * orthogonality correction (convergence is reached when the
182    * difference between two steps of the Frobenius norm of the
183    * correction is below this threshold)
184 
185    * @exception NotARotationMatrixException if the matrix is not a 3X3
186    * matrix, or if it cannot be transformed into an orthogonal matrix
187    * with the given threshold, or if the determinant of the resulting
188    * orthogonal matrix is negative
189 
190    */
191   public Rotation(double[][] m, double threshold)
192     throws NotARotationMatrixException {
193 
194     // dimension check
195     if ((m.length != 3) || (m[0].length != 3)
196         || (m[1].length != 3) || (m[2].length != 3)) {
197       throw new NotARotationMatrixException("a {0}x{1} matrix"
198                                             + " cannot be a rotation matrix",
199                                             new String[] {
200                                               Integer.toString(m.length),
201                                               Integer.toString(m[0].length)
202                                             });
203     }
204 
205     // compute a "close" orthogonal matrix
206     double[][] ort = orthogonalizeMatrix(m, threshold);
207 
208     // check the sign of the determinant
209     double det = ort[0][0] * (ort[1][1] * ort[2][2] - ort[2][1] * ort[1][2])
210                - ort[1][0] * (ort[0][1] * ort[2][2] - ort[2][1] * ort[0][2])
211                + ort[2][0] * (ort[0][1] * ort[1][2] - ort[1][1] * ort[0][2]);
212     if (det < 0.0) {
213       throw new NotARotationMatrixException("the closest orthogonal matrix"
214                                             + " has a negative determinant {0}",
215                                             new String[] {
216                                               Double.toString(det)
217                                             });
218     }
219 
220     // There are different ways to compute the quaternions elements
221     // from the matrix. They all involve computing one element from
222     // the diagonal of the matrix, and computing the three other ones
223     // unsing a formula involving a division by the first element,
224     // which unfortunately can be null. Since the norm of the
225     // quaternion is 1, we know at least one element has an absolute
226     // value greater or equal to 0.5, so it is always possible to
227     // select the right formula and avoid division by zero and even
228     // numerical inaccuracy. Checking the elements in turn and using
229     // the first one greater than 0.45 is safe (this leads to a simple
230     // test since qi = 0.45 implies 4 qi^2 - 1 = -0.19)
231     double s = ort[0][0] + ort[1][1] + ort[2][2];
232     if (s > -0.19) {
233       // compute q0 and deduce q1, q2 and q3
234       q0 = 0.5 * Math.sqrt(s + 1.0);
235       double inv = 0.25 / q0;
236       q1 = inv * (ort[1][2] - ort[2][1]);
237       q2 = inv * (ort[2][0] - ort[0][2]);
238       q3 = inv * (ort[0][1] - ort[1][0]);
239     } else {
240       s = ort[0][0] - ort[1][1] - ort[2][2];
241       if (s > -0.19) {
242         // compute q1 and deduce q0, q2 and q3
243         q1 = 0.5 * Math.sqrt(s + 1.0);
244         double inv = 0.25 / q1;
245         q0 = inv * (ort[1][2] - ort[2][1]);
246         q2 = inv * (ort[0][1] + ort[1][0]);
247         q3 = inv * (ort[0][2] + ort[2][0]);
248       } else {
249         s = ort[1][1] - ort[0][0] - ort[2][2];
250         if (s > -0.19) {
251           // compute q2 and deduce q0, q1 and q3
252           q2 = 0.5 * Math.sqrt(s + 1.0);
253           double inv = 0.25 / q2;
254           q0 = inv * (ort[2][0] - ort[0][2]);
255           q1 = inv * (ort[0][1] + ort[1][0]);
256           q3 = inv * (ort[2][1] + ort[1][2]);
257         } else {
258           // compute q3 and deduce q0, q1 and q2
259           s = ort[2][2] - ort[0][0] - ort[1][1];
260           q3 = 0.5 * Math.sqrt(s + 1.0);
261           double inv = 0.25 / q3;
262           q0 = inv * (ort[0][1] - ort[1][0]);
263           q1 = inv * (ort[0][2] + ort[2][0]);
264           q2 = inv * (ort[2][1] + ort[1][2]);
265         }
266       }
267     }
268 
269   }

271   /*** Build the rotation that transforms a pair of vector into another pair.
272 
273    * <p>Except for possible scale factors, if the instance were applied to
274    * the pair (u<sub>1</sub>, u<sub>2</sub>) it will produce the pair
275    * (v<sub>1</sub>, v<sub>2</sub>).</p>
276 
277    * <p>If the angular separation between u<sub>1</sub> and u<sub>2</sub> is
278    * not the same as the angular separation between v<sub>1</sub> and
279    * v<sub>2</sub>, then a corrected v'<sub>2</sub> will be used rather than
280    * v<sub>2</sub>, the corrected vector will be in the (v<sub>1</sub>,
281    * v<sub>2</sub>) plane.</p>
282 
283    * @param u1 first vector of the origin pair
284    * @param u2 second vector of the origin pair
285    * @param v1 desired image of u1 by the rotation
286    * @param v2 desired image of u2 by the rotation
287    */
288   public Rotation(Vector3D u1, Vector3D u2, Vector3D v1, Vector3D v2) {
289 
290   // norms computation
291   double u1u1 = Vector3D.dotProduct(u1, u1);
292   double u2u2 = Vector3D.dotProduct(u2, u2);
293   double v1v1 = Vector3D.dotProduct(v1, v1);
294   double v2v2 = Vector3D.dotProduct(v2, v2);
295   if ((u1u1 < 1.0e-15) || (u2u2 < 1.0e-15)
296       || (v1v1 < 1.0e-15) || (v2v2 < 1.0e-15))
297     throw new ArithmeticException("null norm");
298 
299   double u1x = u1.getX();
300   double u1y = u1.getY();
301   double u1z = u1.getZ();
302 
303   double u2x = u2.getX();
304   double u2y = u2.getY();
305   double u2z = u2.getZ();
306 
307   // renormalize v1 in order to have (v1'|v1') = (u1|u1)
308   double coeff = Math.sqrt (u1u1 / v1v1);
309   double v1x   = coeff * v1.getX();
310   double v1y   = coeff * v1.getY();
311   double v1z   = coeff * v1.getZ();
312   v1 = new Vector3D(v1x, v1y, v1z);
313 
314   // adjust v2 in order to have (u1|u2) = (v1|v2) and (v2'|v2') = (u2|u2)
315   double u1u2   = Vector3D.dotProduct(u1, u2);
316   double v1v2   = Vector3D.dotProduct(v1, v2);
317   double coeffU = u1u2 / u1u1;
318   double coeffV = v1v2 / u1u1;
319   double beta   = Math.sqrt((u2u2 - u1u2 * coeffU) / (v2v2 - v1v2 * coeffV));
320   double alpha  = coeffU - beta * coeffV;
321   double v2x    = alpha * v1x + beta * v2.getX();
322   double v2y    = alpha * v1y + beta * v2.getY();
323   double v2z    = alpha * v1z + beta * v2.getZ();
324   v2 = new Vector3D(v2x, v2y, v2z);
325 
326   // preliminary computation (we use explicit formulation instead
327   // of relying on the Vector3D class in order to avoid building lots
328   // of temporary objects)
329   Vector3D uRef = u1;
330   Vector3D vRef = v1;
331   double dx1 = v1x - u1.getX();
332   double dy1 = v1y - u1.getY();
333   double dz1 = v1z - u1.getZ();
334   double dx2 = v2x - u2.getX();
335   double dy2 = v2y - u2.getY();
336   double dz2 = v2z - u2.getZ();
337   Vector3D k = new Vector3D(dy1 * dz2 - dz1 * dy2,
338                             dz1 * dx2 - dx1 * dz2,
339                             dx1 * dy2 - dy1 * dx2);
340   double c = k.getX() * (u1y * u2z - u1z * u2y)
341            + k.getY() * (u1z * u2x - u1x * u2z)
342            + k.getZ() * (u1x * u2y - u1y * u2x);
343 
344   if (c < (1.0e-10 * u1u1 * u2u2)) {
345     // the (q1, q2, q3) vector is in the (u1, u2) plane
346     // we try other vectors
347     Vector3D u3 = Vector3D.crossProduct(u1, u2);
348     Vector3D v3 = Vector3D.crossProduct(v1, v2);
349     double u3x  = u3.getX();
350     double u3y  = u3.getY();
351     double u3z  = u3.getZ();
352     double v3x  = v3.getX();
353     double v3y  = v3.getY();
354     double v3z  = v3.getZ();
355     double u3u3 = u1u1 * u2u2 - u1u2 * u1u2;
356 
357     double dx3 = v3x - u3x;
358     double dy3 = v3y - u3y;
359     double dz3 = v3z - u3z;
360     k = new Vector3D(dy1 * dz3 - dz1 * dy3,
361                      dz1 * dx3 - dx1 * dz3,
362                      dx1 * dy3 - dy1 * dx3);
363     c = k.getX() * (u1y * u3z - u1z * u3y)
364       + k.getY() * (u1z * u3x - u1x * u3z)
365       + k.getZ() * (u1x * u3y - u1y * u3x);
366 
367     if (c < (1.0e-10 * u1u1 * u3u3)) {
368       // the (q1, q2, q3) vector is aligned with u1:
369       // we try (u2, u3) and (v2, v3)
370       k = new Vector3D(dy2 * dz3 - dz2 * dy3,
371                        dz2 * dx3 - dx2 * dz3,
372                        dx2 * dy3 - dy2 * dx3);
373       c = k.getX() * (u2y * u3z - u2z * u3y)
374         + k.getY() * (u2z * u3x - u2x * u3z)
375         + k.getZ() * (u2x * u3y - u2y * u3x);
376 
377       if (c < (1.0e-10 * u2u2 * u3u3)) {
378         // the (q1, q2, q3) vector is aligned with everything
379         // this is really the identity rotation
380         q0 = 1.0;
381         q1 = 0.0;
382         q2 = 0.0;
383         q3 = 0.0;
384         return;
385       }
386 
387       // we will have to use u2 and v2 to compute the scalar part
388       uRef = u2;
389       vRef = v2;
390 
391     }
392 
393   }
394 
395   // compute the vectorial part
396   c = Math.sqrt(c);
397   double inv = 1.0 / (c + c);
398   q1 = inv * k.getX();
399   q2 = inv * k.getY();
400   q3 = inv * k.getZ();
401 
402   // compute the scalar part
403    k = new Vector3D(uRef.getY() * q3 - uRef.getZ() * q2,
404                     uRef.getZ() * q1 - uRef.getX() * q3,
405                     uRef.getX() * q2 - uRef.getY() * q1);
406    c = Vector3D.dotProduct(k, k);
407   q0 = Vector3D.dotProduct(vRef, k) / (c + c);
408 
409   }

411   /*** Build one of the rotations that transform one vector into another one.
412 
413    * <p>Except for a possible scale factor, if the instance were
414    * applied to the vector u it will produce the vector v. There is an
415    * infinite number of such rotations, this constructor choose the
416    * one with the smallest associated angle (i.e. the one whose axis
417    * is orthogonal to the (u, v) plane). If u and v are colinear, an
418    * arbitrary rotation axis is chosen.</p>
419 
420    * @param u origin vector
421    * @param v desired image of u by the rotation
422    * @exception ArithmeticException if the norm of one of the vectors is null
423    */
424   public Rotation(Vector3D u, Vector3D v) {
425 
426     double normProduct = u.getNorm() * v.getNorm();
427     if (normProduct < 1.0e-15) {
428       throw new ArithmeticException("null norm");
429     }
430 
431     double dot = Vector3D.dotProduct(u, v);
432 
433     if (dot < ((2.0e-15 - 1.0) * normProduct)) {
434       // special case u = -v: we select a PI angle rotation around
435       // an arbitrary vector orthogonal to u
436       Vector3D w = u.orthogonal();
437       q0 = 0.0;
438       q1 = -w.getX();
439       q2 = -w.getY();
440       q3 = -w.getZ();
441     } else {
442       // general case: (u, v) defines a plane, we select
443       // the shortest possible rotation: axis orthogonal to this plane
444       q0 = Math.sqrt(0.5 * (1.0 + dot / normProduct));
445       double coeff = 1.0 / (2.0 * q0 * normProduct);
446       q1 = coeff * (v.getY() * u.getZ() - v.getZ() * u.getY());
447       q2 = coeff * (v.getZ() * u.getX() - v.getX() * u.getZ());
448       q3 = coeff * (v.getX() * u.getY() - v.getY() * u.getX());
449     }
450 
451   }

453   /*** Build a rotation from three Cardan or Euler elementary rotations.
454 
455    * <p>Cardan rotations are three successive rotations around the
456    * canonical axes X, Y and Z, each axis beeing used once. There are
457    * 6 such sets of rotations (XYZ, XZY, YXZ, YZX, ZXY and ZYX). Euler
458    * rotations are three successive rotations around the canonical
459    * axes X, Y and Z, the first and last rotations beeing around the
460    * same axis. There are 6 such sets of rotations (XYX, XZX, YXY,
461    * YZY, ZXZ and ZYZ), the most popular one being ZXZ.</p>
462    * <p>Beware that many people routinely use the term Euler angles even
463    * for what really are Cardan angles (this confusion is especially
464    * widespread in the aerospace business where Roll, Pitch and Yaw angles
465    * are often wrongly tagged as Euler angles).</p>
466 
467    * @param order order of rotations to use
468    * @param alpha1 angle of the first elementary rotation
469    * @param alpha2 angle of the second elementary rotation
470    * @param alpha3 angle of the third elementary rotation
471    */
472   public Rotation(RotationOrder order,
473                   double alpha1, double alpha2, double alpha3) {
474     Rotation r1 = new Rotation(order.getA1(), alpha1);
475     Rotation r2 = new Rotation(order.getA2(), alpha2);
476     Rotation r3 = new Rotation(order.getA3(), alpha3);
477     Rotation composed = r1.applyTo(r2.applyTo(r3));
478     q0 = composed.q0;
479     q1 = composed.q1;
480     q2 = composed.q2;
481     q3 = composed.q3;
482   }

484   /*** Revert a rotation.
485    * Build a rotation which reverse the effect of another
486    * rotation. This means that if r(u) = v, then r.revert(v) = u. The
487    * instance is not changed.
488    * @return a new rotation whose effect is the reverse of the effect
489    * of the instance
490    */
491   public Rotation revert() {
492     return new Rotation(-q0, q1, q2, q3, false);
493   }

523   /*** Get the normalized axis of the rotation.
524    * @return normalized axis of the rotation
525    */
526   public Vector3D getAxis() {
527     double squaredSine = q1 * q1 + q2 * q2 + q3 * q3;
528     if (squaredSine < 1.0e-12) {
529       return new Vector3D(1, 0, 0);
530     } else if (q0 < 0) {
531       double inverse = 1 / Math.sqrt(squaredSine);
532       return new Vector3D(q1 * inverse, q2 * inverse, q3 * inverse);
533     } else {
534       double inverse = -1 / Math.sqrt(squaredSine);
535       return new Vector3D(q1 * inverse, q2 * inverse, q3 * inverse);
536     }
537   }

539   /*** Get the angle of the rotation.
540    * @return angle of the rotation (between 0 and &pi;)
541    */
542   public double getAngle() {
543     if ((q0 < -0.1) || (q0 > 0.1)) {
544       return 2 * Math.asin(Math.sqrt(q1 * q1 + q2 * q2 + q3 * q3));
545     } else if (q0 < 0) {
546       return 2 * Math.acos(-q0);
547     } else {
548       return 2 * Math.acos(q0);
549     }

552   /*** Get the Cardan or Euler angles corresponding to the instance.
553 
554    * <p>The equations show that each rotation can be defined by two
555    * different values of the Cardan or Euler angles set. For example
556    * if Cardan angles are used, the rotation defined by the angles
557    * a<sub>1</sub>, a<sub>2</sub> and a<sub>3</sub> is the same as
558    * the rotation defined by the angles &pi; + a<sub>1</sub>, &pi;
559    * - a<sub>2</sub> and &pi; + a<sub>3</sub>. This method implements
560    * the following arbitrary choices. For Cardan angles, the chosen
561    * set is the one for which the second angle is between -&pi;/2 and
562    * &pi;/2 (i.e its cosine is positive). For Euler angles, the chosen
563    * set is the one for which the second angle is between 0 and &pi;
564    * (i.e its sine is positive).</p>
565 
566    * <p>Cardan and Euler angle have a very disappointing drawback: all
567    * of them have singularities. This means that if the instance is
568    * too close to the singularities corresponding to the given
569    * rotation order, it will be impossible to retrieve the angles. For
570    * Cardan angles, this is often called gimbal lock. There is
571    * <em>nothing</em> to do to prevent this, it is an intrisic problem
572    * with Cardan and Euler representation (but not a problem with the
573    * rotation itself, which is perfectly well defined). For Cardan
574    * angles, singularities occur when the second angle is close to
575    * -&pi;/2 or +&pi;/2, for Euler angle singularities occur when the
576    * second angle is close to 0 or &pi;, this implies that the identity
577    * rotation is always singular for Euler angles!
578 
579    * @param order rotation order to use
580    * @return an array of three angles, in the order specified by the set
581    * @exception CardanEulerSingularityException if the rotation is
582    * singular with respect to the angles set specified
583    */
584   public double[] getAngles(RotationOrder order)
585     throws CardanEulerSingularityException {
586 
587     final double small        = 1.0e-10;
588     final double maxThreshold = 1.0 - small;
589     final double minThreshold = -maxThreshold;
590 
591     double[] angles = new double[3];
592     Vector3D v1 = null;
593     Vector3D v2 = null;
594 
595     if (order == RotationOrder.XYZ) {
596 
597       // r (Vector3D.plusK) coordinates are :
598       //  sin (theta), -cos (theta) sin (phi), cos (theta) cos (phi)
599       // (-r) (Vector3D.plusI) coordinates are :
600       // cos (psi) cos (theta), -sin (psi) cos (theta), sin (theta)
601       // and we can choose to have theta in the interval [-PI/2 ; +PI/2]
602       v1 = applyTo(Vector3D.plusK);
603       v2 = applyInverseTo(Vector3D.plusI);
604       if  ((v2.getZ() < minThreshold) || (v2.getZ() > maxThreshold)) {
605         throw new CardanEulerSingularityException(true);
606       }
607       angles[0] = Math.atan2(-(v1.getY()), v1.getZ());
608       angles[1] = Math.asin(v2.getZ());
609       angles[2] = Math.atan2(-(v2.getY()), v2.getX());
610 
611     } else if (order == RotationOrder.XZY) {
612 
613       // r (Vector3D.plusJ) coordinates are :
614       // -sin (psi), cos (psi) cos (phi), cos (psi) sin (phi)
615       // (-r) (Vector3D.plusI) coordinates are :
616       // cos (theta) cos (psi), -sin (psi), sin (theta) cos (psi)
617       // and we can choose to have psi in the interval [-PI/2 ; +PI/2]
618       v1 = applyTo(Vector3D.plusJ);
619       v2 = applyInverseTo(Vector3D.plusI);
620       if ((v2.getY() < minThreshold) || (v2.getY() > maxThreshold)) {
621         throw new CardanEulerSingularityException(true);
622       }
623       angles[0] = Math.atan2(v1.getZ(), v1.getY());
624       angles[1] = -Math.asin(v2.getY());
625       angles[2] = Math.atan2(v2.getZ(), v2.getX());
626 
627     } else if (order == RotationOrder.YXZ) {
628 
629       // r (Vector3D.plusK) coordinates are :
630       //  cos (phi) sin (theta), -sin (phi), cos (phi) cos (theta)
631       // (-r) (Vector3D.plusJ) coordinates are :
632       // sin (psi) cos (phi), cos (psi) cos (phi), -sin (phi)
633       // and we can choose to have phi in the interval [-PI/2 ; +PI/2]
634       v1 = applyTo(Vector3D.plusK);
635       v2 = applyInverseTo(Vector3D.plusJ);
636       if ((v2.getZ() < minThreshold) || (v2.getZ() > maxThreshold)) {
637         throw new CardanEulerSingularityException(true);
638       }
639       angles[0] = Math.atan2(v1.getX(), v1.getZ());
640       angles[1] = -Math.asin(v2.getZ());
641       angles[2] = Math.atan2(v2.getX(), v2.getY());
642 
643     } else if (order == RotationOrder.YZX) {
644 
645       // r (Vector3D.plusI) coordinates are :
646       // cos (psi) cos (theta), sin (psi), -cos (psi) sin (theta)
647       // (-r) (Vector3D.plusJ) coordinates are :
648       // sin (psi), cos (phi) cos (psi), -sin (phi) cos (psi)
649       // and we can choose to have psi in the interval [-PI/2 ; +PI/2]
650       v1 = applyTo(Vector3D.plusI);
651       v2 = applyInverseTo(Vector3D.plusJ);
652       if ((v2.getX() < minThreshold) || (v2.getX() > maxThreshold)) {
653         throw new CardanEulerSingularityException(true);
654       }
655       angles[0] = Math.atan2(-(v1.getZ()), v1.getX());
656       angles[1] = Math.asin(v2.getX());
657       angles[2] = Math.atan2(-(v2.getZ()), v2.getY());
658 
659     } else if (order == RotationOrder.ZXY) {
660 
661       // r (Vector3D.plusJ) coordinates are :
662       // -cos (phi) sin (psi), cos (phi) cos (psi), sin (phi)
663       // (-r) (Vector3D.plusK) coordinates are :
664       // -sin (theta) cos (phi), sin (phi), cos (theta) cos (phi)
665       // and we can choose to have phi in the interval [-PI/2 ; +PI/2]
666       v1 = applyTo(Vector3D.plusJ);
667       v2 = applyInverseTo(Vector3D.plusK);
668       if ((v2.getY() < minThreshold) || (v2.getY() > maxThreshold)) {
669         throw new CardanEulerSingularityException(true);
670       }
671       angles[0] = Math.atan2(-(v1.getX()), v1.getY());
672       angles[1] = Math.asin(v2.getY());
673       angles[2] = Math.atan2(-(v2.getX()), v2.getZ());
674 
675     } else if (order == RotationOrder.ZYX) {
676 
677       // r (Vector3D.plusI) coordinates are :
678       //  cos (theta) cos (psi), cos (theta) sin (psi), -sin (theta)
679       // (-r) (Vector3D.plusK) coordinates are :
680       // -sin (theta), sin (phi) cos (theta), cos (phi) cos (theta)
681       // and we can choose to have theta in the interval [-PI/2 ; +PI/2]
682       v1 = applyTo(Vector3D.plusI);
683       v2 = applyInverseTo(Vector3D.plusK);
684       if ((v2.getX() < minThreshold) || (v2.getX() > maxThreshold)) {
685         throw new CardanEulerSingularityException(true);
686       }
687       angles[0] = Math.atan2(v1.getY(), v1.getX());
688       angles[1] = -Math.asin(v2.getX());
689       angles[2] = Math.atan2(v2.getY(), v2.getZ());
690 
691     } else if (order == RotationOrder.XYX) {
692 
693       // r (Vector3D.plusI) coordinates are :
694       //  cos (theta), sin (phi1) sin (theta), -cos (phi1) sin (theta)
695       // (-r) (Vector3D.plusI) coordinates are :
696       // cos (theta), sin (theta) sin (phi2), sin (theta) cos (phi2)
697       // and we can choose to have theta in the interval [0 ; PI]
698       v1 = applyTo(Vector3D.plusI);
699       v2 = applyInverseTo(Vector3D.plusI);
700       if ((v2.getX() < minThreshold) || (v2.getX() > maxThreshold)) {
701         throw new CardanEulerSingularityException(false);
702       }
703       angles[0] = Math.atan2(v1.getY(), -v1.getZ());
704       angles[1] = Math.acos(v2.getX());
705       angles[2] = Math.atan2(v2.getY(), v2.getZ());
706 
707     } else if (order == RotationOrder.XZX) {
708 
709       // r (Vector3D.plusI) coordinates are :
710       //  cos (psi), cos (phi1) sin (psi), sin (phi1) sin (psi)
711       // (-r) (Vector3D.plusI) coordinates are :
712       // cos (psi), -sin (psi) cos (phi2), sin (psi) sin (phi2)
713       // and we can choose to have psi in the interval [0 ; PI]
714       v1 = applyTo(Vector3D.plusI);
715       v2 = applyInverseTo(Vector3D.plusI);
716       if ((v2.getX() < minThreshold) || (v2.getX() > maxThreshold)) {
717         throw new CardanEulerSingularityException(false);
718       }
719       angles[0] = Math.atan2(v1.getZ(), v1.getY());
720       angles[1] = Math.acos(v2.getX());
721       angles[2] = Math.atan2(v2.getZ(), -v2.getY());
722 
723     } else if (order == RotationOrder.YXY) {
724 
725       // r (Vector3D.plusJ) coordinates are :
726       //  sin (theta1) sin (phi), cos (phi), cos (theta1) sin (phi)
727       // (-r) (Vector3D.plusJ) coordinates are :
728       // sin (phi) sin (theta2), cos (phi), -sin (phi) cos (theta2)
729       // and we can choose to have phi in the interval [0 ; PI]
730       v1 = applyTo(Vector3D.plusJ);
731       v2 = applyInverseTo(Vector3D.plusJ);
732       if ((v2.getY() < minThreshold) || (v2.getY() > maxThreshold)) {
733         throw new CardanEulerSingularityException(false);
734       }
735       angles[0] = Math.atan2(v1.getX(), v1.getZ());
736       angles[1] = Math.acos(v2.getY());
737       angles[2] = Math.atan2(v2.getX(), -v2.getZ());
738 
739     } else if (order == RotationOrder.YZY) {
740 
741       // r (Vector3D.plusJ) coordinates are :
742       //  -cos (theta1) sin (psi), cos (psi), sin (theta1) sin (psi)
743       // (-r) (Vector3D.plusJ) coordinates are :
744       // sin (psi) cos (theta2), cos (psi), sin (psi) sin (theta2)
745       // and we can choose to have psi in the interval [0 ; PI]
746       v1 = applyTo(Vector3D.plusJ);
747       v2 = applyInverseTo(Vector3D.plusJ);
748       if ((v2.getY() < minThreshold) || (v2.getY() > maxThreshold)) {
749         throw new CardanEulerSingularityException(false);
750       }
751       angles[0] = Math.atan2(v1.getZ(), -v1.getX());
752       angles[1] = Math.acos(v2.getY());
753       angles[2] = Math.atan2(v2.getZ(), v2.getX());
754 
755     } else if (order == RotationOrder.ZXZ) {
756 
757       // r (Vector3D.plusK) coordinates are :
758       //  sin (psi1) sin (phi), -cos (psi1) sin (phi), cos (phi)
759       // (-r) (Vector3D.plusK) coordinates are :
760       // sin (phi) sin (psi2), sin (phi) cos (psi2), cos (phi)
761       // and we can choose to have phi in the interval [0 ; PI]
762       v1 = applyTo(Vector3D.plusK);
763       v2 = applyInverseTo(Vector3D.plusK);
764       if ((v2.getZ() < minThreshold) || (v2.getZ() > maxThreshold)) {
765         throw new CardanEulerSingularityException(false);
766       }
767       angles[0] = Math.atan2(v1.getX(), -v1.getY());
768       angles[1] = Math.acos(v2.getZ());
769       angles[2] = Math.atan2(v2.getX(), v2.getY());
770 
771     } else { // last possibility is ZYZ
772 
773       // r (Vector3D.plusK) coordinates are :
774       //  cos (psi1) sin (theta), sin (psi1) sin (theta), cos (theta)
775       // (-r) (Vector3D.plusK) coordinates are :
776       // -sin (theta) cos (psi2), sin (theta) sin (psi2), cos (theta)
777       // and we can choose to have theta in the interval [0 ; PI]
778       v1 = applyTo(Vector3D.plusK);
779       v2 = applyInverseTo(Vector3D.plusK);
780       if ((v2.getZ() < minThreshold) || (v2.getZ() > maxThreshold)) {
781         throw new CardanEulerSingularityException(false);
782       }
783       angles[0] = Math.atan2(v1.getY(), v1.getX());
784       angles[1] = Math.acos(v2.getZ());
785       angles[2] = Math.atan2(v2.getY(), -v2.getX());
786 
787     }
788 
789     return angles;
790

793   /*** Get the 3X3 matrix corresponding to the instance
794    * @return the matrix corresponding to the instance
795    */
796   public double[][] getMatrix() {
797 
798     // products
799     double q0q0  = q0 * q0;
800     double q0q1  = q0 * q1;
801     double q0q2  = q0 * q2;
802     double q0q3  = q0 * q3;
803     double q1q1  = q1 * q1;
804     double q1q2  = q1 * q2;
805     double q1q3  = q1 * q3;
806     double q2q2  = q2 * q2;
807     double q2q3  = q2 * q3;
808     double q3q3  = q3 * q3;
809 
810     // create the matrix
811     double[][] m = new double[3][];
812     m[0] = new double[3];
813     m[1] = new double[3];
814     m[2] = new double[3];
815 
816     m [0][0] = 2.0 * (q0q0 + q1q1) - 1.0;
817     m [1][0] = 2.0 * (q1q2 - q0q3);
818     m [2][0] = 2.0 * (q1q3 + q0q2);
819 
820     m [0][1] = 2.0 * (q1q2 + q0q3);
821     m [1][1] = 2.0 * (q0q0 + q2q2) - 1.0;
822     m [2][1] = 2.0 * (q2q3 - q0q1);
823 
824     m [0][2] = 2.0 * (q1q3 - q0q2);
825     m [1][2] = 2.0 * (q2q3 + q0q1);
826     m [2][2] = 2.0 * (q0q0 + q3q3) - 1.0;
827 
828     return m;
829

832   /*** Apply the rotation to a vector.
833    * @param u vector to apply the rotation to
834    * @return a new vector which is the image of u by the rotation
835    */
836   public Vector3D applyTo(Vector3D u) {
837 
838     double x = u.getX();
839     double y = u.getY();
840     double z = u.getZ();
841 
842     double s = q1 * x + q2 * y + q3 * z;
843 
844     return new Vector3D(2 * (q0 * (x * q0 - (q2 * z - q3 * y)) + s * q1) - x,
845                         2 * (q0 * (y * q0 - (q3 * x - q1 * z)) + s * q2) - y,
846                         2 * (q0 * (z * q0 - (q1 * y - q2 * x)) + s * q3) - z);
847 
848   }

869   /*** Apply the instance to another rotation.
870    * Applying the instance to a rotation is computing the composition
871    * in an order compliant with the following rule : let u be any
872    * vector and v its image by r (i.e. r.applyTo(u) = v), let w be the image
873    * of v by the instance (i.e. applyTo(v) = w), then w = comp.applyTo(u),
874    * where comp = applyTo(r).
875    * @param r rotation to apply the rotation to
876    * @return a new rotation which is the composition of r by the instance
877    */
878   public Rotation applyTo(Rotation r) {
879     return new Rotation(r.q0 * q0 - (r.q1 * q1 + r.q2 * q2 + r.q3 * q3),
880                         r.q1 * q0 + r.q0 * q1 + (r.q2 * q3 - r.q3 * q2),
881                         r.q2 * q0 + r.q0 * q2 + (r.q3 * q1 - r.q1 * q3),
882                         r.q3 * q0 + r.q0 * q3 + (r.q1 * q2 - r.q2 * q1),
883                         false);

905   /*** Perfect orthogonality on a 3X3 matrix.
906    * @param m initial matrix (not exactly orthogonal)
907    * @param threshold convergence threshold for the iterative
908    * orthogonality correction (convergence is reached when the
909    * difference between two steps of the Frobenius norm of the
910    * correction is below this threshold)
911    * @return an orthogonal matrix close to m
912    * @exception NotARotationMatrixException if the matrix cannot be
913    * orthogonalized with the given threshold after 10 iterations
914    */
915   private double[][] orthogonalizeMatrix(double[][] m, double threshold)
916     throws NotARotationMatrixException {
917     double x00 = m[0][0];
918     double x01 = m[0][1];
919     double x02 = m[0][2];
920     double x10 = m[1][0];
921     double x11 = m[1][1];
922     double x12 = m[1][2];
923     double x20 = m[2][0];
924     double x21 = m[2][1];
925     double x22 = m[2][2];
926     double fn = 0;
927     double fn1;
928 
929     double[][] o = new double[3][];
930     o[0] = new double[3];
931     o[1] = new double[3];
932     o[2] = new double[3];
933 
934     // iterative correction: Xn+1 = Xn - 0.5 * (Xn.Mt.Xn - M)
935     int i = 0;
936     while (++i < 11) {
937 
938       // Mt.Xn
939       double mx00 = m[0][0] * x00 + m[1][0] * x10 + m[2][0] * x20;
940       double mx10 = m[0][1] * x00 + m[1][1] * x10 + m[2][1] * x20;
941       double mx20 = m[0][2] * x00 + m[1][2] * x10 + m[2][2] * x20;
942       double mx01 = m[0][0] * x01 + m[1][0] * x11 + m[2][0] * x21;
943       double mx11 = m[0][1] * x01 + m[1][1] * x11 + m[2][1] * x21;
944       double mx21 = m[0][2] * x01 + m[1][2] * x11 + m[2][2] * x21;
945       double mx02 = m[0][0] * x02 + m[1][0] * x12 + m[2][0] * x22;
946       double mx12 = m[0][1] * x02 + m[1][1] * x12 + m[2][1] * x22;
947       double mx22 = m[0][2] * x02 + m[1][2] * x12 + m[2][2] * x22;
948 
949       // Xn+1
950       o[0][0] = x00 - 0.5 * (x00 * mx00 + x01 * mx10 + x02 * mx20 - m[0][0]);
951       o[0][1] = x01 - 0.5 * (x00 * mx01 + x01 * mx11 + x02 * mx21 - m[0][1]);
952       o[0][2] = x02 - 0.5 * (x00 * mx02 + x01 * mx12 + x02 * mx22 - m[0][2]);
953       o[1][0] = x10 - 0.5 * (x10 * mx00 + x11 * mx10 + x12 * mx20 - m[1][0]);
954       o[1][1] = x11 - 0.5 * (x10 * mx01 + x11 * mx11 + x12 * mx21 - m[1][1]);
955       o[1][2] = x12 - 0.5 * (x10 * mx02 + x11 * mx12 + x12 * mx22 - m[1][2]);
956       o[2][0] = x20 - 0.5 * (x20 * mx00 + x21 * mx10 + x22 * mx20 - m[2][0]);
957       o[2][1] = x21 - 0.5 * (x20 * mx01 + x21 * mx11 + x22 * mx21 - m[2][1]);
958       o[2][2] = x22 - 0.5 * (x20 * mx02 + x21 * mx12 + x22 * mx22 - m[2][2]);
959 
960       // correction on each elements
961       double corr00 = o[0][0] - m[0][0];
962       double corr01 = o[0][1] - m[0][1];
963       double corr02 = o[0][2] - m[0][2];
964       double corr10 = o[1][0] - m[1][0];
965       double corr11 = o[1][1] - m[1][1];
966       double corr12 = o[1][2] - m[1][2];
967       double corr20 = o[2][0] - m[2][0];
968       double corr21 = o[2][1] - m[2][1];
969       double corr22 = o[2][2] - m[2][2];
970 
971       // Frobenius norm of the correction
972       fn1 = corr00 * corr00 + corr01 * corr01 + corr02 * corr02
973         + corr10 * corr10 + corr11 * corr11 + corr12 * corr12
974         + corr20 * corr20 + corr21 * corr21 + corr22 * corr22;
975 
976       // convergence test
977       if (Math.abs(fn1 - fn) <= threshold)
978         return o;
979 
980       // prepare next iteration
981       x00 = o[0][0];
982       x01 = o[0][1];
983       x02 = o[0][2];
